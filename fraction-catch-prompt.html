✨ Game Prompt
Create a simple HTML5 fraction game. Performance is the top priority — it must run smoothly on a 2018 iPad (A10 chip) inside this platform's code preview window. YOUR RESPONSE FORMAT: Introduce the game in 2–3 sentences ONLY. No long explanations. Do NOT explain how the code works. Do NOT include instructions to copy, save, or open the file — the student will use the preview button on this platform. Go straight to the code block. Keep the code as short and compact as possible. STRICT PERFORMANCE RULES: Use a single HTML canvas element for ALL rendering — no moving DOM elements, no CSS animations, no image files, no emoji. Draw everything with simple shapes (rectangles, circles) and fillText for fractions. Maximum 3 falling objects on screen at any time. Use requestAnimationFrame but cap to ~30fps using a delta-time check. Keep everything in ONE HTML file — inline JS, inline CSS, no external libraries. Use touchstart and touchmove events for input (not click). GAME DESIGN — "Fraction Catch": A basket (simple rectangle) sits at the bottom. The player drags it left/right by touch. Coloured circles fall slowly from the top, appearing one at a time. Never more than 3 on screen. Each circle shows a fraction (e.g. 3/4, 1/6, 5/8). A target is shown at the top: "Catch fractions LARGER than 1/2" or "Catch fractions EQUAL to 3/4" (including equivalent fractions like 6/8). Correct catch = +1 point. Wrong catch = −1 point. Missing a correct one = no penalty. Every 5 catches, the target changes. Use large, clear text (minimum 28px). FRACTIONS TO USE: Only use denominators from: {2, 3, 4, 5, 6, 8, 10, 12}. All fractions should be proper fractions or equal to 1. Include equivalent fractions (e.g. 2/4 and 3/6 both appear alongside 1/2). VISUAL STYLE: Light blue background (#e0f4ff). Falling circles in 3–4 bright solid colours. Basket is a simple dark rectangle. Score and target displayed clearly at the top in large text. Correct catch: flash score green briefly. Wrong catch: flash red. Canvas must fill the screen and work in portrait orientation.
